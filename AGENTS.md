# AGENTS.md

## プロジェクトの目的

inkduetは、人とAIが二人で一つの記事を仕上げる記事作成ワークスペースである。
修正専用IDEやAIによる自動代筆ではなく、伝えたいことからストーリーを設計し、調査、執筆、
レビュー、修正を経て、人間が完成を判断するまでを一貫して扱う。

## 正本

- プロダクトの目的、MVP、成功条件、判断履歴: `docs/product-brief.md`
- 編集・収束工程の課題と機能バックログ: `docs/editorial-workflow.md`
- Codex App Server、Skill、アプリの責務分担: `docs/app-server-architecture.md`

同じ内容を複数の文書へ重複させない。判断が変わった場合は、該当する正本に
「日付、Before、After、理由」を追記する。

## プロダクト原則

- 書き手が主導し、AIは共作者として支援する。人間の承認を推測しない。
- 一つの記事を意図から完成まで途切れさせない。
- ストーリー、本文、レビュー、変更、確認状態の対応を失わない。
- 専用アプリの価値は生成能力ではなく、人間の比較・選択・承認を短くすることに置く。
- Skillが記事制作の知識を、アプリが状態遷移と判断操作を所有し、二重管理しない。

## 実装時の規約

- JavaScript / TypeScriptのパッケージ管理にはpnpmを使い、`packageManager`へ正確な
  バージョンを記録し、`pnpm-lock.yaml`を含める。
- Codex App Serverのプロトコル型は、固定したCodexバージョンから生成する。
- 同梱Skillを明示的に呼び出し、利用者のローカル環境にあるSkillや暗黙発動へ依存しない。
- 外部公開、削除、認証情報の変更など不可逆または対外的な操作は、人間の明示承認なしに行わない。
